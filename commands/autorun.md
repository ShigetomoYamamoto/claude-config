# /autorun — 自走モード（ハードストップ付き）

完了条件を1つ渡すと、`rules/loop-safety.md` の安全条件を満たした上で、その条件を満たすまで人間のプロンプトなしで作業を継続する。

**このコマンドの原則:** ブレーキのないループは事故。走り出す前に必ず安全条件を確認し、ハードストップを設定する。

## ステップ 1: 起動前チェック（loop-safety.md の前提条件）

以下の4条件をすべて満たすか確認する。1つでも欠ければ自走を始めず、不足を報告して止まる。

1. **完了条件が一文で言え、機械検証できるか**（例: `pytest が全 pass し ruff check がクリーン`）
2. **専用ブランチ or worktree で作業しているか**（`main` / `master` / `develop` では自走しない）
3. **機械的な成功判定があるか**（テスト・lint・型チェック・スクリプト。エージェントの自己申告は不可）
4. **ハードストップが設定されているか**（ターン / 時間 / トークンのいずれか）

ユーザーが上限を指定していなければ「**20ターン or 30分の早い方**」をデフォルトとして提示し、合意を取る。

## ステップ 2: 手段の選択

完了条件と起動契機に応じて使い分ける:

| 状況 | 使う仕組み |
|------|-----------|
| 単一の完了条件まで自走（テストが通るまで等） | 組み込み `/goal <条件> or stop after <N> turns` |
| 一定間隔で繰り返し起動（CI 監視・PR 巡回等） | `/loop <interval> <command>` または `ScheduleWakeup` |
| 決まった時刻・cron で起動 | `CronCreate`（schedule スキル） |
| 多数の独立タスクをファンアウトして検証 | `Workflow`（pipeline / parallel） |

## ステップ 3: 自走

1. **完了条件の定式化** — 機械検証可能な形に落とす。Claude 自身の出力で証明できる条件にする（評価器は会話出力しか見ない）。
2. **ハードストップの設定** — ターン / 時間 / トークンのいずれかを明示的に組み込む。
3. **開始** — ステップ 2 で選んだ手段で走らせる。
4. **各反復で完了条件を再確認** — 当初の条件からズレたら（ゴールドリフト）、新しい目標を勝手に作らず STOP して報告する。
5. **不可逆操作の前で停止** — `git push` / deploy / delete / 外部送信は、ループ中でも人間に確認する。前の反復の承認は次に引き継がない。

## ステップ 4: 停止と報告

完了条件の達成、またはハードストップ到達で停止し、以下を報告する:

- 達成 / 未達
- 消費したターン数・時間（・概算トークン）
- 残課題（未達の場合）
